# Disposition Combat UX Overhaul — Design v1.0

**Дата:** 2026-03-02
**Статус:** Утверждён
**Цель:** Сделать бой понятным с первого хода, убрать визуальный шум, починить технические баги, добавить juice.
**Референсы:** Gwent (простота восприятия), Tainted Grail (easy to learn, hard to master)
**Принцип:** Действие простое — глубина в последствиях.

---

## 1. Сохраняем

- **Disposition Track** (-100…+100) с двумя путями победы (уничтожить / подчинить)
- **Три действия:** Strike, Influence, Sacrifice
- **Engine:** DispositionCombatSimulation, EnemyAI, momentum, fate deck, resonance zones
- **Determinism:** seed-based, воспроизводимый

---

## 2. Меняем: Схема экрана

### До (текущий)
- 10+ информационных слоёв одного визуального веса
- Физические зоны (Strike/Influence/Sacrifice) как Y-полосы на экране
- Drag карты в зону
- Мелкие карты (0.65x), перегруженный HUD

### После (новый)

```
┌──────────────────────────────────────┐
│  Turn 3          ♥ 85    ⚡ 2/3      │  ← Компактный HUD (одна строка)
├──────────────────────────────────────┤
│                                      │
│          ┌──────────────┐            │
│          │   IDOL (враг)│            │  ← Враг крупнее, центр внимания
│          │  ⚔ ATK 5     │            │  ← Intent прямо на враге
│          └──────────────┘            │
│                                      │
│  ◄ Уничтожить ═══════●══════ Подчинить ►│  ← DISPOSITION BAR
│    -100        -23              +100 │     большой, с подписями
│                                      │
├──────────────────────────────────────┤
│                                      │
│     [Карта 1] [Карта 2] [Карта 3]    │  ← Рука: крупные, читаемые
│                                      │
│  ── при тапе на карту: ────────────  │
│                                      │
│   ⚔ Удар     ☽ Влияние   ♦ Жертва   │  ← 3 кнопки-действия
│    −8           +5         +1⚡      │     с предпросмотром числа
│                                      │
│              [Конец хода]            │
└──────────────────────────────────────┘
```

### Ключевые принципы:
1. **HUD — одна строка** (Turn, HP, Energy). Всё остальное скрыто.
2. **Зоны → кнопки действий.** Появляются при выборе карты. Показывают итоговое число.
3. **Disposition bar — центральный элемент.** Большой, с человекочитаемыми подписями.
4. **Карты крупнее.** Без мелкого визуального шума.
5. **Второстепенная инфа по long-press:**
   - Long-press на карту → fate keyword, streak bonus, resonance modifier, формула
   - Long-press на HUD → discard/exhaust count, resonance zone, modifier stack, статистика

## 3. Меняем: Interaction flow

### До
1. Drag карту → попади в Y-зону → непонятно что будет

### После
1. **TAP на карту** → карта поднимается, 3 кнопки действий появляются снизу
2. На кнопках — **итоговые числа** (все модификаторы учтены):
   - Обычное число → белый/жёлтый
   - Ослабленное (adapt/provoke) → красный цвет
   - Усиленное (streak/surge) → зелёный с ↑
3. **TAP на кнопку** → карта летит к disposition bar, маркер сдвигается
4. **TAP на другую карту** → предыдущая возвращается, числа обновляются
5. **Drag** остаётся как альтернативный input (перетащить карту на кнопку)
6. **End Turn** → кнопка всегда видна

### Что это даёт:
- Предсказуемость: игрок ВИДИТ число до действия
- Простота: tap-tap вместо drag + aim
- Обучаемость: цветные числа подсказывают оптимальное действие
- Скорость: 2 тапа вместо drag

## 4. Меняем: Обратная связь (Juiciness)

### При розыгрыше карты:
- Карта анимированно летит к disposition bar (0.3с)
- Floating text с числом сдвига (красное = Strike, синее = Influence)
- Маркер disposition bar анимированно сдвигается
- Fate keyword → flash-уведомление (1с): "Прилив! ×1.5 силы"
- Haptic: medium (обычное), heavy (критический сдвиг >15)

### При ходе врага:
- Враг "пульсирует" (0.3с anticipation) перед действием
- Floating text у HUD: "Атака! −5 ♥" (красный)
- HP в HUD мигает красным при уроне
- Defend/Provoke/Adapt → бейдж на враге с числом
- Haptic: light (defend/adapt), heavy (атака)

### При смене режима врага:
- Аура меняется (уже есть)
- Flash-текст: "Враг в отчаянии!" (1с)
- Первый раз — развёрнутое пояснение (2с): "Атаки усилены вдвое!"

### При завершении боя:
- Экран результата (перед переходом):
```
┌──────────────────────────┐
│      ⚔ УНИЧТОЖЕН        │
│                          │
│  Ходов:        5         │
│  Карт сыграно: 12        │
│  Итог:   −100 ███████    │
│  HP героя: 85/100        │
│                          │
│     [Продолжить]         │
└──────────────────────────┘
```

## 5. Технические фиксы (параллельно с UI)

### Критические (блокируют прохождение):
1. **fatalError() → graceful error** в `ViewModel.makeCombatResult()` — вместо крэша возвращать ошибку если `outcome == nil`
2. **Верифицировать EnemyActionResolver** — убедиться что враг реально наносит урон герою (HP вычитается)
3. **Очищать adaptPenalty** после применения — сейчас стакается бесконечно
4. **Сбрасывать enemyModeStrikeBonus** при выходе из survival mode
5. **Подключить enemy mode bonus к EnemyAI** — сейчас вычисляется но не применяется

### Визуальные фиксы:
6. Disposition bar: корректное отображение на крайних значениях (-100/+100)
7. Hand size: фиксировать startingHandSize один раз при старте боя

## 6. Информационная иерархия

| Уровень | Содержимое | Как видно |
|---------|-----------|-----------|
| **Всегда видно** | HP, Energy, Turn, Disposition bar, Enemy intent, Hand | На экране |
| **При выборе карты** | 3 кнопки с итоговыми числами, цветовая подсказка | Появляется при тапе |
| **По long-press на карту** | Fate keyword, streak bonus, resonance modifier, формула расчёта | Popup overlay |
| **По long-press на HUD** | Discard/exhaust count, resonance zone, modifier stack, статистика боя | Popup overlay |
| **По long-press на врага** | Enemy mode, vulnerabilities, текущие модификаторы | Popup overlay |

Принцип: ~60% текущего визуального шума убрано с экрана, но 100% информации доступно.

---

## 7. Scope и ограничения

### В scope:
- Перестройка layout (компактный HUD, крупные карты, кнопки действий)
- Новый interaction flow (tap-tap с предпросмотром)
- Обратная связь (floating text, flash уведомления, анимации)
- Экран результата боя
- 5 критических технических фиксов
- Информационная иерархия (long-press popups)

### Вне scope:
- Изменение engine механик (disposition, momentum, fate deck, enemy modes)
- Новые карточные способности
- Онбординг/туториал (следующий этап)
- Save/resume боя (следующий этап)
- Sound effects (используем существующие callbacks)

## 8. Риски

| Риск | Митигация |
|------|-----------|
| Кнопки действий перегружают нижнюю часть экрана | Кнопки появляются только при выборе карты, исчезают после действия |
| Long-press popups неочевидны для нового игрока | Первый раз показывать hint "нажми и удерживай для деталей" |
| Потеря тактильности drag | Drag остаётся как альтернативный input |
| Расчёт итоговых чисел для предпросмотра — сложная логика | Использовать существующий DispositionCalculator, добавить preview метод |
